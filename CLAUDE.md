# Инструкции по работе с Claude Code

## Структура проекта

Это сайт компании SK-Rosa (ремонтные услуги). Основные директории:
- `src/` - исходные файлы
- `src/pages/` - HTML страницы
- `src/assets/` - статические ресурсы (CSS, JS, изображения)
- `public_html/` - готовая сборка для продакшена
- `scripts/` - скрипты сборки и генерации

## Как экономить токены

### 1. Давай конкретные задачи
**Плохо:** "Исправь сайт"
**Хорошо:** "Исправь ссылку в footer на странице src/pages/plastering.html"

### 2. Указывай путь к файлу
**Плохо:** "Измени заголовок на главной"
**Хорошо:** "Измени h1 в src/index.html на 'Новый заголовок'"

### 3. Используй контекст IDE
- Открой нужный файл перед запросом - я его увижу
- Выдели нужный код - я пойму что менять

### 4. Группируй похожие задачи
**Плохо:** Отдельные запросы на каждый файл
**Хорошо:** "Добавь телефон +7-999-123-45-67 в footer всех страниц в src/pages/plastering/"

### 5. Используй шаблоны
Если нужно создать много похожих страниц - попроси создать скрипт генератора, а не делать вручную.

## Полезные команды

```bash
# Сборка проекта
npm run build

# Локальный сервер
npm run dev

# Превью готовой сборки
npm run preview
```

## Правила работы с проектом

### Стиль кода
- HTML страницы на русском языке
- Используем Vite для сборки
- CSS в отдельных файлах, не inline

### Git
- Коммиты на русском языке
- Перед коммитом: `npm run build` для проверки

### Изображения
- Формат WebP предпочтителен
- Размещать в `src/assets/images/`

## Частые задачи

### Создать новую городскую страницу
1. Скопировать шаблон из существующей страницы
2. Заменить название города
3. Добавить ссылку в навигацию на where-we-work.html

### Добавить статью в блог
1. Создать файл в `src/pages/articles/`
2. Добавить ссылку в `src/pages/blog.html`

### Изменить цены
- Цены указаны в HTML страницах услуг
- Также проверить калькулятор: `src/pages/calculator.html`

## Что НЕ нужно делать

- Не создавай README.md без запроса
- Не добавляй комментарии в код без необходимости
- Не рефактори код, если не просили
- Не меняй структуру проекта самовольно
